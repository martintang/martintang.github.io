ZSH和oh-my-zsh

发表于 2017-06-18 | 分类于 技术笔记 |

在Mac里，ZSH是个经常被使用的Shell，很大原因是有一个非常优秀和方便的配置框架oh-my-zsh。关于ZSH和oh-my-zsh使用的一些技巧在这里做个汇总，方便以后查询。

ZSH相关的配置有:

  1. 环境变量ZDOTDIR，决定了ZSH去哪找个人配置文件，默认是个人目录，以下描述都以ZDOTDIR是默认值为例。
  2. .zshenv，ZSH首先读取/etc/zshenv，然后便是~/.zshenv。一般，.zshenv放置环境变量相关的设置。
  3. .zprofile，如果shell是login shell，则ZSH先读取/etc/zprofile，然后读取~/.zprofile。
  4. .zshrc，如果shell是interactive shell，则ZSH先读取/etc/zshrc，然后读取~/.zshrc。
  5. .zlogin，如果shell是login shell，则ZSH先读取/etc/zlogin，然后读取~/.zlogin。
  6. .zlogout，当用户登出时，ZSH先读取/etc/zlogout，然后读取~/.zlogout。

oh-my-zsh个性化配置:

  1. 环境变量ZSH_CUSTOM，指定了oh-my-zsh的个人配置文件夹，默认是~/.oh-my-zsh/custom。
  2. 自定义plugin，在$ZSH_CUSTOM/plugins下创建目录plugin_name，然后在目录plugin_name下创建script，必须以.plugin.zsh结尾。自定义的plugin可以是新建的，也可以覆盖oh-my-zsh自带的plugin，覆盖的时候，plugin_name和script_name和要覆盖的plugin同名。
  3. 自定义theme，在$ZSH_CUSTOM/themes下放入自定义的theme文件，必须以.zsh-theme结尾，如my_awesome_theme.zsh-theme，然后在.zshrc里指定该theme: ZSH_THEME="my_awesome_theme"，
    同理，该方法也可以覆盖已有的theme。
  4. 自定义function，在$ZSH_CUSTOM下创建文件，以.zsh结尾，然后自定义function即可。如果要覆盖lib/*.zsh，则在$ZSH_CUSTOM/lib下创建同名文件进行覆盖。
  5. ZSH_CUSTOM目录可以进行版本控制，不会有任何影响。
