Mac OSX10.10 Android源码下载及编译(4.4.4)

发表于 2015-08-25 | 分类于 技术笔记 |

因为工作原因，需要下载Android源码进行研究，本以为用Mac下载源码编译是个很容易的事情，实际上，碰到了很多问题，特记录之。

Android源码下载

主要参考Android官网，然后一些自己碰到问题的解决：

  1. 准备一个能连接国外的代理，推荐Astrill和Lantern，速度都很快也很稳定。
    这2个代理都是针对web的，要在shell中下载源码，还得设置环境变量：
    export https_proxy=127.0.0.1:xxxx
    [注]：如果代理被封或者速度太慢，考虑同步、更新、下载Android Source & SDK from 国内镜像站
  2. 因为Android源码编译是要区分大小写的，而Mac本身的文件系统不区分大小写，所以得在Mac上建一个"区分大小写，日志式"的磁盘。
    下载4.4.4源码要42G，编译后out文件夹大小22G，所以总共需要64G空间，先分配70G空间的磁盘：
    hdiutil create -type SPARSE -fs 'Case-sensitive Journaled HFS+' -size 70g ~/android.dmg
    或
    用磁盘工具Disk Utility来做一个70G大小的磁盘
    未来如果磁盘空间不足，可以增大磁盘空间：
    hdiutil resize -size <new-size-you-want>g ~/android.dmg
    然后，挂载磁盘：
    hdiutil attach ~/android.dmg -mountpoint /Volumes/android
    创建下载目录：
    mkdir /Volumes/android/android-source
  3. 下载工具repo
    curl https://storage.googleapis.com/git-repo-downloads/repo > /usr/local/bin/repo
    chmod a+x /usr/local/bin/repo
  4. 修改文件描述符限制，在~/.zshrc中添加以下内容：
    ulimit -S -n 1024
  5. 初始化repo client：
    cd /Volumes/android/android-source
    repo init -u https://android.googlesource.com/platform/manifest -b android-4.4.4_r2.0.1
    [注]：-b指定了branch，要想换个branch，详见Codenames, Tags, and Build Numbers
  6. 在下载过程中，有时候经常会卡住，虽然可以重新repo sync，但是会发现源码占用空间不断增大，曾经源码占用空间增长到50G，后来发现问题所在[1]，所以做一些额外的准备工作，都是经验教训：
    • 将Mac的节能器设置为防止电脑自动进入睡眠，防止硬盘进入睡眠。
    • tcp window scaling disable[2]
      sudo sysctl -w net.inet.tcp.rfc1323=0
    • 准备自动重试下载脚本download.sh[3]，在重试下载的时候要把临时文件清理掉，防止占用空间不断增大。
      find . -name "tmp_pack*" | xargs rm
  7. 运行download.sh开始下载Android源码，不用坐在电脑前等卡住了再按repo sync了。

Android源码编译

源码编译过程会碰到一些问题：

  1. 下载JDK6，编译4.4及以前的代码需要java1.6版本，4.4以后的可以用java1.7版本。
  2. 修改系统框架源码，直接注释QTMovieModernizer.h:156, 163两行的atomic关键字,需要修改头文件如下:
    /Applications/Xcode.app/Contents/Developer/Platforms/MacOSX.platform/Developer/SDKs/MacOSX10.9.sdk/System/Library/Frameworks/QTKit.framework/Headers/QTMovieModernizer.h

  3. 修改一些android源码，使其能够在10.10系统下编译[4][5]：

    • build/core/combo/HOST_darwin-x86.mk

    • external/chromium_org/base/android/jni_generator/jni_generator.py

    • external/qemu/distrib/sdl-1.2.15/src/video/quartz/SDL_QuartzVideo.h

  4. 移除NDK_ROOT环境变量
    unset NDK_ROOT

  5. 编译
    make -j4

Android源码版本切换

当我们需要切换到别的版本编译时，需要做如下操作：

  1. 初始化repo客户端到某一版本(以android-4.0.3_r1为例)
    repo init -b android-4.0.3_r1
  2. 同步代码
    repo sync
  3. 查询现在在哪个版本
    repo info后看Current revision
  4. 如果需要构造本地分支，用如下命令
    repo start my-local-branch --all

[注]：要在别的版本下编译，应该会碰到一些别的问题，需要上网查找解决方案[6]。
